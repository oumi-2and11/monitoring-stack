服务说明

这是本项目的自研 Flask 被测服务，使用 prometheus_client 库暴露自定义业务指标（Counter + Histogram），供 Prometheus 采集并在 Grafana 中可视化。

通过以下接口可以模拟不同类型的请求，观察监控系统的采集与告警效果：

GET /
正常响应接口，返回 200 状态码。用于模拟正常的业务请求流量。
200 OK
GET /error
模拟服务器错误，返回 500 状态码。用于触发 FlaskAppErrorRateHigh 告警（错误率 > 5% 持续 1 分钟）。
500 Error
GET /slow
模拟慢请求，随机延迟 0.5~3.0 秒后返回 200。用于观察延迟 P50/P95/P99 分位数变化。
200 Slow
GET /metrics
Prometheus 指标端点，暴露 flask_http_request_total（Counter）和 flask_http_request_duration_seconds（Histogram）。
Prometheus

指标说明

flask_http_request_total（Counter）— 请求总数，按 method / endpoint / status_code 分类标签

flask_http_request_duration_seconds（Histogram）— 请求延迟分布，支持 P50 / P95 / P99 分位数计算

Histogram 桶边界: [5ms, 10ms, 25ms, 50ms, 100ms, 250ms, 500ms, 1s, 2.5s, 5s]

相关监控入口

Prometheus Targets: http://localhost:9090/targets
Prometheus Alerts: http://localhost:9090/alerts
Grafana Dashboard: http://localhost:3000
Alert Receiver: http://localhost:5001
