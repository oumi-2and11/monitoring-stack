# 项目使用说明

## 环境要求

- Windows 10/11
- Docker Desktop（WSL2 后端），确保 Docker 引擎正在运行
- PowerShell 5.1+（Windows 自带）

## 一、启动项目

打开 PowerShell，进入项目目录：

```powershell
cd monitoring-stack
docker compose up -d --build
```

首次运行需要拉取镜像并构建 Flask 服务，约 2-5 分钟。后续启动只需几秒。

等待所有容器启动后，确认状态：

```powershell
docker compose ps
```

应看到 7 个容器全部 `Up`。

也可使用一键脚本：

```powershell
.\scripts\up.ps1
```

## 二、访问各服务

| 服务 | 地址 | 说明 |
|---|---|---|
| Grafana | http://localhost:3000 | 登录账号：admin / admin |
| Prometheus | http://localhost:9090 | 直接访问，无需登录 |
| Alertmanager | http://localhost:9093 | 直接访问 |
| Flask 被测服务 | http://localhost:8000 | 接口：`/` `/error` `/slow` `/metrics` |
| 告警接收日志 | http://localhost:5001 | 查看已收到的告警记录 |
| cAdvisor | http://localhost:8080 | 容器指标 UI |
| Node Exporter | http://localhost:9100/metrics | 主机指标原始数据 |

## 三、查看监控面板

1. 打开 http://localhost:3000，输入 admin / admin 登录
2. 左侧菜单点击 **Dashboards**
3. 可看到两个自动加载的 Dashboard：
   - **Host & Docker Overview**：主机 CPU/内存/磁盘/网络 + 容器 CPU Top5/内存 Top5/重启次数
   - **Flask Service Overview**：请求 QPS、错误率、延迟 P50/P95/P99、告警状态

> 数据源和 Dashboard 均通过 Provisioning 自动配置，无需手动操作。

## 四、触发告警演示

### 方式一：使用脚本

```powershell
# 拉高 5xx 错误率（持续约 2 分钟）
.\scripts\trigger_error.ps1
```

### 方式二：手动 curl

```powershell
# 循环调用 /error 接口
for ($i=0; $i -lt 500; $i++) { Invoke-RestMethod http://localhost:8000/error -ErrorAction SilentlyContinue }
```

### 观察告警全链路

1. **Prometheus** → http://localhost:9090/alerts，`FlaskAppErrorRateHigh` 从绿色 INACTIVE 变为红色 FIRING
2. **Alertmanager** → http://localhost:9093，出现 active 告警
3. **告警接收日志** → http://localhost:5001，表格显示告警名、级别、实例、时间
4. **Grafana Dashboard** → Flask Service Overview 中错误率 Gauge 上升，Alert Status 变为 FIRING

停止触发后约 1-2 分钟，告警自动恢复（RESOLVED）。

## 五、动态服务发现演示

```powershell
# 向 targets.json 追加一个新目标
.\scripts\add_target.ps1 -Target "prometheus:9090"
```

然后访问 http://localhost:9090/targets，等待约 30 秒刷新，可看到新 target 自动出现。

## 六、压测（可选）

项目内置 Locust 压测脚本，需先安装 Locust：

```powershell
# 使用项目虚拟环境（推荐）
.venv\Scripts\python.exe -m pip install locust
```

运行压测：

```powershell
# 无界面模式，30 并发，持续 90 秒
.venv\Scripts\python.exe -m locust -f loadtest/locustfile.py --host=http://localhost:8000 --headless -u 30 -r 5 -t 90s --only-summary
```

压测期间打开 Grafana 观察实时数据变化，结果可截图写入报告。

## 七、停止项目

```powershell
# 停止但保留数据
docker compose down

# 或使用脚本
.\scripts\down.ps1

# 停止并清除所有数据（下次启动如全新环境）
.\scripts\down.ps1 -Clean
```

## 常见问题

### 端口冲突

如果某端口被占用，修改 `docker-compose.yml` 中对应服务的端口映射，左侧为宿主机端口：

```yaml
ports:
  - "3000:3000"   # 改左侧数字，如 "3001:3000"
```

### Docker Desktop 未启动

运行命令时报 `open //./pipe/dockerDesktopLinuxEngine` 错误，说明 Docker Desktop 未运行。从开始菜单启动 Docker Desktop，等待托盘图标显示 running 后重试。

### cAdvisor 指标不完整

cAdvisor 在 Windows Docker Desktop (WSL2) 下部分 cgroup 指标可能缺失，这是已知限制。容器级别的 CPU/内存指标通常正常。

### Grafana 忘记密码

```powershell
docker compose down -v
docker compose up -d
```

清除 volume 后重启，Grafana 恢复默认 admin/admin。
