## 2. 作业要求逐条"对标"怎么满足

### (1) "搭建 Prometheus+Grafana 全方位监控告警系统"

你用 docker compose 起：

- Prometheus（采集 + rules）
- Grafana（展示）
- Alertmanager（告警通知）
- windows_exporter（主机指标，适配 Windows 环境）
- cAdvisor（Docker 容器指标）
- Flask app（你自己写的服务指标）
- alert_receiver（你自己写的 webhook 接收器，用于演示告警落地）
- Nginx（反向代理，安全加固入口）

> 这条是工程落地的主干。

------

### (2) "静态 + 动态服务发现"

- **静态发现**：prometheus.yml 里写死 targets（例如 windows_exporter、cadvisor、flask_app）
- **动态发现（稳妥版强推 file_sd_configs）**：
	- Prometheus 支持从一个 JSON 文件读取 targets �件表
	- 你把 `prometheus/file_sd/targets.json` 提交进去
	- 你写一个小脚本来"改 targets.json 并让 Prometheus 自动刷新"，这就叫动态发现

> 评分点：你在报告里解释为什么容器环境必须动态发现、file_sd 的工作机制和优缺点。

------

### (3) "对容器、物理节点、service、pod 等资源指标监控"

落地做法：

- **物理节点（Windows 宿主机）**：windows_exporter（CPU、内存、磁盘、网络、服务状态）
	- 报告里补充说明：Linux 环境对应 node_exporter，两者指标命名有差异（如 `windows_cpu_time_total` vs `node_cpu_seconds_total`），PromQL 需适配
- **容器**：cAdvisor（每个容器 CPU/内存/网络/重启等）
	- 注意：cAdvisor 在 Windows Docker Desktop (WSL2) 下可能无法获取完整 cgroup 指标，报告里说明此限制
- **service（你的服务）**：Flask + prometheus_client（QPS、错误率、延迟）
- **pod**：这一项用**研究为辅**完成：
	- 报告里写 K8s 下如何用 kube-state-metrics、cAdvisor、kubelet、metrics-server 采集 pod 指标
	- 给出 Prometheus 在 K8s 的 service discovery（基于 Kubernetes API）怎么配置
	- 附"如果有环境，可用 minikube 复现"的步骤（可选）

> 老师一般会认可这种"Docker落地 + K8s路径清晰可复现"的完成方式，尤其你是单人。

### (5) "研究 docker 与 kubernetes 编排环境下的安全监控"

这部分**报告 + 实操**都要做：

#### 报告部分

- **监控系统本身的安全风险**：
	- /metrics 暴露信息（进程、版本、路径等），可能造成信息泄露
	- Prometheus/Grafana/Alertmanager 默认无强认证，暴露端口的风险
	- 解决：网络隔离（只在内网/compose网络）、反向代理 + BasicAuth、TLS、Grafana 权限、最小暴露端口
- **容器/编排环境的安全监控思路**：
	- 异常重启次数、异常网络流量、错误率突增、CPU/内存异常飙升作为安全事件"间接指标"
	- 用告警规则做初步检测（不是替代 IDS，但能做 SRE+安全联动）

#### 实操部分（安全课高分关键）

- **Nginx 反向代理 + BasicAuth**：
	- 所有服务（Prometheus/Grafana/Alertmanager）不直接暴露端口，统一走 Nginx 入口
	- Nginx 配置 `auth_basic`，未认证访问返回 401
	- 演示：无密码 curl 被拒 → 带密码 curl 成功 → 截图写进报告
- **Prometheus /metrics 端点加密**：
	- 通过 Nginx 代理 Prometheus，要求 BasicAuth 才能访问
	- 报告里对比"暴露 vs 加固"的安全差异
- **Grafana 权限控制**：
	- 配置 Admin + Viewer 两个角色，演示最小权限原则
- **Docker 网络隔离**：
	- docker-compose.yml 里只用内部网络 `monitoring-net`，只对 Nginx 开放宿主机端口
	- 报告里画网络拓扑图，标注哪些端口对外、哪些仅内部

> 这是网络空间安全课的作业，安全实操的权重比普通课程高。光写理论不够，必须有"加固前 vs 加固后"的演示对比。

### (6) "研究 Prometheus 告警工具包 + K8s下部署"

- 告警工具包就是 Prometheus rules + Alertmanager 路由/接收器
- K8s 部署写清楚两条路线：
	1. Helm 安装 kube-prometheus-stack（最常见）
	2. Prometheus Operator 的 CRD 方式（ServiceMonitor/PodMonitor）
- 你可以画一张"Docker版 vs K8s版"对比表（采集方式、服务发现、运维复杂度、安全边界）

#### 告警规则（具体内容，至少配以下 5 条）

| 规则名 | 表达式 | 阈值 | 持续时间 | 级别 |
|---|---|---|---|---|
| HostHighCpuLoad | `100 - (avg by(instance)(rate(windows_cpu_time_total{mode="idle"}[2m])) * 100)` | > 80% | 1m | warning |
| HostHighMemoryUsage | `100 - (100 * windows_os_physical_memory_free_bytes / windows_cs_physical_memory_bytes)` | > 90% | 2m | warning |
| FlaskAppErrorRateHigh | `rate(flask_http_request_duration_seconds_count{status_code=~"5.."}[1m]) / rate(flask_http_request_duration_seconds_count[1m])` | > 5% | 1m | critical |
| ContainerHighCpu | `container_cpu_usage_seconds_total` 按容器聚合 | > 80% | 2m | warning |
| ContainerRestartCount | `increase(container_restart_count[10m])` | > 3 | 0m | critical |

## 3. Windows 环境的特别注意（避免踩坑）

1. **建议使用 Docker Desktop（WSL2 后端）**，这样 cAdvisor 挂载路径更稳定
2. cAdvisor 在 Windows 上有时对宿主机指标不如 Linux 完整，但监控"容器层面"通常够用
3. **使用 windows_exporter 替代 node_exporter**：
	- windows_exporter 是 node_exporter 的 Windows 版本，指标更准确
	- 部署方式：在 Docker Compose 中用容器版，或直接在宿主机安装 MSI
	- 报告里说明两者差异，体现"因地制宜"（加分点）
4. 所有配置里 targets 用容器服务名（如 `prometheus:9090`、`cadvisor:8080`），不要写 `localhost`
5. **数据持久化**：docker-compose.yml 必须配置 volumes，防止容器重启后数据丢失：
	- `prometheus_data:/prometheus`
	- `grafana_data:/var/lib/grafana`
	- `alertmanager_data:/alertmanager`

## 5. Docker Compose 服务与网络拓扑（安全隔离）

```
┌─────────────────────────────────────────────────┐
│                   宿主机                          │
│                                                   │
│   :80/443  ──►  Nginx (反向代理 + BasicAuth)      │
│                    │                               │
│         ┌──────────┼──────────┐                    │
│         ▼          ▼          ▼                    │
│   Prometheus   Grafana   Alertmanager             │
│     :9090       :3000      :9093                   │
│         │          │          │                    │
│         ▼          ▼          ▼                    │
│     ┌─── monitoring-net (内部网络，不对外) ───┐     │
│     │                                          │     │
│     │  windows_exporter  cAdvisor  flask_app   │     │
│     │  alert_receiver                         │     │
│     └──────────────────────────────────────────┘     │
│                                                   │
└─────────────────────────────────────────────────┘
```

- 只有 Nginx 对外暴露端口（80/443）
- 其余服务全部在 `monitoring-net` 内部网络
- 报告里用这张图说明"最小暴露面"的安全设计原则
