## 阶段一：基础骨架

### 开发内容

1. 创建 `docker-compose.yml`，定义 prometheus（9090）+ grafana（3000）
2. 创建 `prometheus/prometheus.yml`，最简配置：scrape_interval 15s，自监控 job
3. 创建 `grafana/provisioning/datasources/datasource.yml`，自动注册 Prometheus 数据源
4. 配置 volumes 持久化

### 验证方法

```powershell
docker compose up -d
# 1. http://localhost:9090/targets，prometheus target UP
# 2. http://localhost:3000，Grafana 登录 admin/admin
# 3. Data Sources 中 Prometheus 数据源已自动配置
# 4. Explore 查询 up{job="prometheus"} 有结果
docker compose down
```

---

## 阶段二：采集层

### 开发内容

1. **node_exporter**（端口 9100）— 主机 CPU/内存/磁盘/网络
2. **cAdvisor**（端口 8080）— 容器 CPU/内存/网络/重启
3. **Flask 应用**（端口 8000）— Counter（请求总数、错误数）+ Histogram（延迟）
   - 接口：`/`（正常）、`/error`（500）、`/slow`（慢请求）、`/metrics`

### 验证方法

```powershell
docker compose up -d --build
# 1. http://localhost:9090/targets，三个新 target 全部 UP
# 2. http://localhost:8000/metrics，flask_http_request_* 指标可见
# 3. Prometheus 查询 node_cpu_seconds_total、container_cpu_usage_seconds_total 有数据
docker compose down
```

## 阶段三：服务发现

### 开发内容

1. **静态发现**：prometheus.yml 中的 static_configs（阶段二已完成）
2. **动态发现（file_sd_configs）**：
   - `prometheus/file_sd/targets.json` 放 flask_app 目标
   - prometheus.yml 新增 `flask_app_sd` job，refresh_interval 30s
   - `scripts/add_target.ps1` 动态追加目标

### 验证方法

```powershell
docker compose up -d
# 1. http://localhost:9090/targets，flask_app_sd job 出现且 UP
# 2. 运行 add_target.ps1 追加目标
# 3. 等待 30s，刷新 /targets，新 target 自动出现
docker compose down
```

## 阶段四：告警体系

### 开发内容

1. **5 条告警规则**：HostHighCpuLoad、HostHighMemoryUsage、FlaskAppErrorRateHigh、ContainerHighCpu、ContainerRestartCount
2. **Alertmanager**（端口 9093）：按 job 分组，webhook 推送到 alert_receiver
3. **alert_receiver**（端口 5001）：Flask 接收 POST /webhook，GET / 展示告警表格
4. **触发脚本**：trigger_error.ps1、trigger_latency.ps1

### 验证方法

```powershell
docker compose up -d --build
# 1. http://localhost:9090/rules，5 条规则全部加载
# 2. 运行 trigger_error.ps1 约 2 分钟
# 3. http://localhost:9090/alerts，FlaskAppErrorRateHigh 变为 FIRING
# 4. http://localhost:9093，Alertmanager 出现告警
# 5. http://localhost:5001，alert_receiver 显示告警 JSON
docker compose down
```

## 阶段五：可视化

### 开发内容

1. **Dashboard 1：Host & Docker 概览**（7 个 Panel：CPU/内存/磁盘/网络/容器CPU Top5/容器内存 Top5/重启次数）
2. **Dashboard 2：Flask 服务概览**（6 个 Panel：QPS/错误率 Gauge/总请求数/延迟P50-P95-P99/延迟按端点/告警状态）
3. **Provisioning**：datasource.yml + providers.yml + 挂载 dashboards 目录，自动加载

### 验证方法

```powershell
docker compose down -v && docker compose up -d
# 1. http://localhost:3000，2 个 Dashboard 自动出现
# 2. 所有 Panel 有数据
docker compose down
```

## 阶段七：压测与验收

### 开发内容

1. **Locust 压测**：locustfile.py，80% 访问 /，10% /error，10% /slow
2. **启动/停止脚本**：up.ps1、down.ps1（-Clean 清除数据）

### 验证方法

```powershell

.venv/Scripts/python.exe -m locust -f loadtest/locustfile.py --host=http://localhost:8000 --headless -u 30 -r 5 -t 90s --only-summary

# 观察 Prometheus /alerts 告警状态
# 观察 alert_receiver 收到告警

# 全链路验收
.\scripts\up.ps1
# 1. docker compose ps 确认全部 running
# 2. http://localhost:9090/targets 全部 UP
# 3. http://localhost:3000 两个 Dashboard 已加载
# 4. 运行 trigger_error.ps1，告警全链路触发
.\scripts\down.ps1
```
