# 接口文档

## 总览

| 服务 | 地址 | 说明 |
|---|---|---|
| Prometheus | http://localhost:9090 | 采集引擎，提供查询/告警/配置等 API |
| Grafana | http://localhost:3000 | 可视化面板，admin/admin 登录 |
| Alertmanager | http://localhost:9093 | 告警路由与通知 |
| Flask 被测服务 | http://localhost:8000 | 自研服务，提供业务接口与指标 |
| Alert Receiver | http://localhost:5001 | 自研告警接收器，展示告警记录 |
| cAdvisor | http://localhost:8080 | 容器指标 UI |
| Node Exporter | http://localhost:9100/metrics | 主机指标原始数据 |

## 1. Prometheus（端口 9090）

### 1.1 Web UI 页面

| 路径 | 方法 | 说明 |
|---|---|---|
| `/` | GET | Prometheus 主页，可输入 PromQL 执行即时查询 |
| `/graph` | GET | 图表查询页面，支持即时查询和范围查询，可绘制折线图 |
| `/targets` | GET | 采集目标列表，显示所有 scrape job 的健康状态（UP/DOWN）、最后抓取时间、错误信息 |
| `/alerts` | GET | 告警规则列表，显示每条规则当前状态（INACTIVE / PENDING / FIRING） |
| `/rules` | GET | 告警规则详情，显示规则文件、表达式、阈值等配置 |
| `/config` | GET | 当前生效的 prometheus.yml 完整配置 |
| `/status` | GET | 运行状态信息：启动时间、TSDB 统计、Wal 检查点等 |
| `/flags` | GET | Prometheus 启动命令行参数 |
| `/service-discovery` | GET | 服务发现状态，显示每个 job 的发现目标和已采集目标的映射关系 |

### 1.2 API 接口

| 路径 | 方法 | 参数 | 说明 |
|---|---|---|---|
| `/api/v1/query` | GET/POST | `query`（PromQL 表达式）, `time`（时间戳，可选） | 即时查询，返回某一时刻的查询结果 |
| `/api/v1/query_range` | GET/POST | `query`, `start`, `end`, `step` | 范围查询，返回一段时间内的数据序列 |
| `/api/v1/labels` | GET/POST | — | 列出所有标签名 |
| `/api/v1/label/<name>/values` | GET | — | 列出指定标签的所有值 |
| `/api/v1/series` | GET/POST | `match[]`（选择器） | 列出匹配的时间序列 |
| `/api/v1/alerts` | GET | — | 返回所有告警的当前状态 |
| `/api/v1/targets` | GET | — | 返回所有采集目标的状态（含 labels、lastScrape、health） |

### 1.3 采集目标

| Job 名称 | 发现方式 | 目标地址 | 采集路径 | 采集间隔 |
|---|---|---|---|---|
| `prometheus` | static_configs | `localhost:9090` | `/metrics` | 15s |
| `node_exporter` | static_configs | `node_exporter:9100` | `/metrics` | 15s |
| `cadvisor` | static_configs | `cadvisor:8080` | `/metrics` | 15s |
| `flask_app` | static_configs | `flask_app:8000` | `/metrics` | 15s |
| `flask_app_sd` | file_sd_configs | `targets.json` | `/metrics` | 15s（刷新 30s） |

## 3. Alertmanager（端口 9093）

### 3.1 Web UI 页面

| 路径 | 方法 | 说明 |
|---|---|---|
| `/` | GET | 告警列表，显示当前 active 和 silenced 的告警 |
| `/alerts` | GET | 同上，告警列表主页面 |
| `/status` | GET | Alertmanager 运行状态和集群信息 |

### 3.2 API 接口

| 路径 | 方法 | 说明 |
|---|---|---|
| `/api/v1/alerts` | GET | 获取当前所有告警 |
| `/api/v1/alerts` | POST | 创建告警（由 Prometheus 调用） |
| `/api/v1/silences` | GET | 获取静默规则列表 |
| `/api/v1/silences` | POST | 创建静默规则 |
| `/api/v1/status` | GET | 获取运行状态 |

### 3.3 告警路由配置

| 配置项 | 值 | 说明 |
|---|---|---|
| group_by | `["alertname", "job"]` | 按告警名和 job 分组 |
| group_wait | `10s` | 新组首次等待 10 秒发送通知 |
| group_interval | `30s` | 同组新增告警 30 秒后发送 |
| repeat_interval | `5m` | 重复通知间隔 5 分钟 |
| receiver | `webhook` | 默认推送到 webhook |
| resolve_timeout | `5m` | 告警恢复超时 5 分钟 |

## 4. Flask 被测服务（端口 8000）

### 4.1 业务接口

| 路径 | 方法 | 状态码 | 说明 |
|---|---|---|---|
| `/` | GET | 200 | 正常响应，返回服务说明页面 |
| `/error` | GET | 500 | 模拟服务器错误，用于触发错误率告警 |
| `/slow` | GET | 200 | 模拟慢请求，随机延迟 0.5~3.0 秒 |
| `/metrics` | GET | 200 | Prometheus 指标端点，纯文本格式 |

#### GET /

- **响应**：HTML 页面，展示服务说明和接口导航
- **指标记录**：`flask_http_request_total{method="GET",endpoint="/",status_code="200"} +1`
- **延迟记录**：`flask_http_request_duration_seconds{method="GET",endpoint="/"}`

#### GET /error

- **响应**：HTML 页面（HTTP 500），说明此接口的作用和触发方式
- **指标记录**：`flask_http_request_total{method="GET",endpoint="/error",status_code="500"} +1`
- **延迟记录**：`flask_http_request_duration_seconds{method="GET",endpoint="/error"}`

#### GET /slow

- **响应**：HTML 页面，显示本次请求的实际延迟秒数
- **指标记录**：`flask_http_request_total{method="GET",endpoint="/slow",status_code="200"} +1`
- **延迟记录**：`flask_http_request_duration_seconds{method="GET",endpoint="/slow"}`（含 sleep 时间）

#### GET /metrics

- **响应**：Prometheus exposition format 纯文本，Content-Type: `text/plain; version=0.0.4; charset=utf-8`
- **主要指标**：

| 指标名 | 类型 | 标签 | 说明 |
|---|---|---|---|
| `flask_http_request_total` | Counter | method, endpoint, status_code | 请求总数 |
| `flask_http_request_duration_seconds_bucket` | Histogram | method, endpoint, le | 延迟桶计数 |
| `flask_http_request_duration_seconds_sum` | Histogram | method, endpoint | 延迟总和 |
| `flask_http_request_duration_seconds_count` | Histogram | method, endpoint | 延迟观测次数 |
| `python_gc_objects_collected_total` | Counter | generation | Python GC 统计 |
| `process_cpu_seconds_total` | Counter | — | 进程 CPU 时间 |
| `process_resident_memory_bytes` | Gauge | — | 进程驻留内存 |
| `process_start_time_seconds` | Gauge | — | 进程启动时间 |

## 5. Alert Receiver（端口 5001）

### 5.1 接口

| 路径 | 方法 | Content-Type | 说明 |
|---|---|---|---|
| `/` | GET | text/html | 告警记录展示页面，表格显示所有已接收告警 |
| `/webhook` | POST | application/json | 接收 Alertmanager 推送的告警数据 |

#### POST /webhook

Alertmanager 按 `send_resolved: true` 配置，同时推送 FIRING 和 RESOLVED 事件。

请求体示例：

```json
{
  "receiver": "webhook",
  "status": "firing",
  "alerts": [
    {
      "status": "firing",
      "labels": {
        "alertname": "FlaskAppErrorRateHigh",
        "job": "flask_app",
        "instance": "flask_app:8000",
        "severity": "critical"
      },
      "annotations": {
        "summary": "Flask app error rate is high",
        "description": "5xx error rate is 85.2% (threshold 5%)"
      },
      "startsAt": "2026-05-14T10:00:00Z",
      "endsAt": "0001-01-01T00:00:00Z"
    }
  ],
  "groupLabels": { "alertname": "FlaskAppErrorRateHigh", "job": "flask_app" },
  "commonLabels": { "severity": "critical" },
  "externalURL": "http://alertmanager:9093"
}
```

#### GET /

展示页字段说明：

| 字段 | 来源 | 说明 |
|---|---|---|
| Time | 服务端接收时间 | 告警到达 Alert Receiver 的时间 |
| Status | `alert.status` | `firing`（红色）或 `resolved`（绿色） |
| Alert | `alert.labels.alertname` | 告警规则名称 |
| Severity | `alert.labels.severity` | `warning`（黄色）或 `critical`（红色） |
| Instance | `alert.labels.instance` | 触发告警的实例地址 |
| Summary | `alert.annotations.summary` | 告警摘要 |

## 7. Node Exporter（端口 9100）

| 路径 | 方法 | 说明 |
|---|---|---|
| `/metrics` | GET | 主机指标，Prometheus exposition format 纯文本 |

Node Exporter 没有独立 UI 页面，仅提供 `/metrics` 端点供 Prometheus 采集。暴露约 284 个 `node_*` 前缀指标，涵盖 CPU、内存、磁盘、网络、文件系统等。

## 8. 告警规则

| 规则名 | 表达式 | 阈值 | 持续时间 | 级别 | 关联指标来源 |
|---|---|---|---|---|---|
| HostHighCpuLoad | `100 - avg by(instance)(rate(node_cpu_seconds_total{mode="idle"}[2m])) * 100` | > 80% | 1m | warning | node_exporter |
| HostHighMemoryUsage | `100 - 100 * node_memory_MemAvailable_bytes / node_memory_MemTotal_bytes` | > 90% | 2m | warning | node_exporter |
| FlaskAppErrorRateHigh | `sum by(job,instance)(rate(flask_http_request_total{status_code=~"5.."}[1m])) / sum by(job,instance)(rate(flask_http_request_total[1m])) * 100` | > 5% | 1m | critical | flask_app |
| ContainerHighCpu | `rate(container_cpu_usage_seconds_total{...}[2m]) * 100` | > 80% | 2m | warning | cAdvisor |
| ContainerRestartCount | `increase(container_restart_count[10m])` | > 3 | 0m | critical | cAdvisor |

## 9. 告警全链路

```
Flask /error 接口被大量调用
        │
        ▼
Prometheus 采集 flask_http_request_total 指标
        │
        ▼
告警规则 FlaskAppErrorRateHigh 评估：错误率 > 5% 且持续 1m
        │
        ▼
状态从 INACTIVE → PENDING → FIRING
        │
        ├─→ Prometheus /alerts 页面显示红色 FIRING
        ├─→ Grafana Alert Status 面板显示红色 "FIRING"
        │
        ▼
Prometheus 推送告警到 Alertmanager:9093
        │
        ├─→ Alertmanager / 页面显示 active 告警
        │
        ▼
Alertmanager 按 job 分组后推送到 webhook
        │
        ▼
Alert Receiver:5001 接收并展示告警记录
```

错误率恢复后（停止触发脚本约 1-2 分钟），告警自动变为 RESOLVED，Alert Receiver 同样收到 resolved 事件。
